# 포인터

## 개요

22/07/17

📎 변수와 포인터

- 변수: 특정 데이터 값을 가지고 있음.
- 포인터: 특정 데이터가 저장된 기억 장소의 주소 값을 가지고 있음.
- 포인터는 기억 공간을 변수명으로 접근하지 않고 주소로 접근하기 위해 사용

📎 포인터의 특징

- 일반적인 변수의 기억 공간 표현
- 주소를 이용하여 변수와 같은 동일한 작업 가능

📎 일반 변수

- 정수형 변수 a 선언
    
    ```c
    int a;
    ```
    
- 변수 a에 100 할당
    
    ```c
    a=100;
    ```
    
- 변수 a의 주소
    
    ```c
    &a
    ```
    
- 주소 연산 불가능

📎 포인터 변수

- 포인터 변수 a를 정수형으로 선언
    
    ```c
    int *a;
    ```
    
- a 주소에 100 할당
    
    ```c
    *a=100;
    ```
    
- a 자체가 주소
    
    ```c
    a
    ```
    
- 주소 연산 가능

📎 포인터 변수 선언

- 변수 선언
    
    ```c
    int a, b;
    ```
    
- 변수 p를 포인터 변수로 선언
    
    ```c
    int *p;
    ```
    
- 포인터 변수 p에 변수 a의 주소 값을 대입; a의 번지 수가 p에 저장됨. 값 X
    
    ```c
    a = 5000;
    p = &a;
    ```
    
- 포인터 p가 가리키는 주소의 내용을 변수 b에 저장; 번지 수에 해당하는 값이 저장됨.
    
    ```c
    b = *p;
    ```
    

## 포인터 변수

📎 void형 포인터

- 프로그램 실행시에 자료형이 결정되는 경우에 사용
- 저장하기 전에 명시적 형변환이 필요

📎 void형 포인터 예

- void형 포인터 선언 후 초기화
    
    ```c
    int a = 100;
    char b = 'b';
    void *p = NULL;
    ```
    
- void형 포인터 p에 int형 변수 a의 주소를 명시적 형변환을 이용하여 대입
    
    ```c
    p = (int*)&a; //100
    ```
    
- void형 포인터 p에 char형 변수 b의 주소를 명시적 형변환을 이용하여 대입
    
    ```c
    p = (char*)&b; //b
    ```
    

📎 char 포인터

- 포인터는 문자열 처리에 효과적
- 문자열 처리에 char형 포인터 사용
- 문자열 포인터는 문자열의 시작 주소를 가짐;
    
    ```c
    char *cp = "COMPUTER";
    ```

📎 이중 포인터

- 자료가 있는 곳을 이중으로 가리키는 포인터
- 이중 포인터가 가리키는 주소에는 자료가 아닌 주소 값이 들어 있고, 그 주소에 자료가 들어 있음.
- 포인터 변수에 다시 포인터를 지정하는 것으로서, 포인터에 대한 포인터

📎 이중 포인터 예

- p; 기억 공간 주소
- *p; 주소 p에 수록되어 있는 값으로 이 값이 주소로 사용됨.
- **p; 주소가 *p인 곳에 수록되어 있는 값
    
    ```c
    int **p;
    ```
